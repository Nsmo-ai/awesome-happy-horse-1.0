Un dépôt public de veille et de ressources autour de [Happy Horse AI](https://tryhappyhorse.com/), structuré pour rendre l’information plus vérifiable et plus réutilisable.

## Ce que ce dépôt apporte

- des faits actuellement vérifiables
- un registre de claims avec `Verified / Likely / Rumor / Outdated`
- une chronologie expliquant pourquoi [Happy Horse 1.0](https://tryhappyhorse.com/) est devenu visible si vite
- une lecture pratique des benchmarks publics
- des cas de prompt utiles pour évaluer le modèle
- des cas d’échec et des alertes de fiabilité

## Points clés

- Au `2026-04-13`, le dépôt public [`happyhorseai/happyhorse`](https://github.com/happyhorseai/happyhorse) existe, mais sa surface publique reste légère.
- Le README public actuel ressemble davantage à une page marketing qu’à une publication technique.
- Au `2026-04-13`, [Happy Horse 1.0](https://tryhappyhorse.com/) apparaît très haut dans plusieurs catégories publiques d’Artificial Analysis.
- Aucune preuve publique n’indique que des poids ou du code d’inférence aient été publiés dans le dépôt officiel.
- Ce dépôt normalise le point d’entrée utilisateur vers [Happy Horse](https://tryhappyhorse.com/).
